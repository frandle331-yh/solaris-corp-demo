# WordPress 組み込み用ファイル(SOLARIS 子テーマ)

`index.html` / `news.html` / `news-article.html`(静的デモ)を、WordPressで
「クライアント自身がお知らせを投稿・更新できる」形に実装するための子テーマ一式です。

## 構成

```
wp-theme/solaris-child/
├─ style.css          … 子テーマ定義(Template: 親テーマ名)+ 調整用CSS
├─ functions.php      … お知らせ用カスタム投稿タイプ「news」+ カテゴリー(news_category)を登録
├─ header.php         … 共通ヘッダー(ナビ・ロゴ)
├─ footer.php         … 共通フッター
├─ front-page.php     … トップページ(お知らせ最新4件を自動表示)
├─ archive-news.php   … お知らせ一覧(/news/・カテゴリー絞り込み・ページ送り)
└─ single-news.php    … お知らせ記事詳細
```

## この構成が実現すること(今回のご依頼に対応)

- **NEWSページの追加**: `/news/` に一覧、各記事に詳細ページが自動生成されます。
- **クライアント自身での投稿・更新**: 管理画面の左メニューに「お知らせ」が追加され、
  ブロックエディタ(Gutenberg)で**専門知識なしに記事の追加・編集・公開・削除**ができます。
  カテゴリー(お知らせ / プレスリリース / 採用情報 など)も管理画面から自由に増やせます。
- **トップページへの自動反映**: お知らせを追加すると、トップの「お知らせ」欄にも最新4件が自動で反映されます。
- **既存サイトを壊さない**: 親テーマを直接編集せず**子テーマ**として追加するため、
  既存のデザイン・データに影響を与えずにカスタマイズできます。

## 導入手順(概略)

1. 既存テーマを親テーマとして、本 `solaris-child` を `wp-content/themes/` に配置
   (`style.css` の `Template:` を実際の親テーマのフォルダ名に合わせます)。
2. 管理画面 → 外観 → テーマ から子テーマを有効化。
3. 「設定 → パーマリンク」を一度保存(お知らせのURLを有効化)。
4. 「お知らせ」メニューからカテゴリーを作成し、記事を投稿。
5. 「設定 → 表示設定」でトップページを固定ページ(front-page)に設定。

## 補足

- 文章修正・デザイン調整は、実案件では既存テーマの構造に合わせて対応します。
- 静的デモの見た目(`assets/style.css`)を子テーマの `style.css` に集約すれば、
  デザインを保ったままWordPress上で動作します。
- お問い合わせフォームは Contact Form 7 等のプラグインで追加可能です。
